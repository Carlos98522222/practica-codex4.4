# Actividad: Propuesta de mini proyecto asistida con IA

## Objetivo
Diseñar y documentar una propuesta de **proyecto pequeño** implementable en una sola iteración corta, usando una tecnología permitida:
- ARM64 Assembly
- C
- Python
- Bash

## Resultado esperado
Al finalizar, cada estudiante deberá entregar:
1. Una propuesta clara del problema a resolver.
2. Un caso de uso principal con entradas/salidas definidas.
3. Una estructura de repositorio coherente.
4. Un plan básico de pruebas.
5. Un prototipo mínimo funcional en `src/main.py` (o equivalente si cambias de lenguaje).

## Restricciones académicas
- Mantén el alcance pequeño (máximo 1-2 funcionalidades principales).
- No usar APIs pagadas.
- No usar bases de datos, nube ni contenedores.
- Debe ser ejecutable con herramientas gratuitas de IA y software local.

## Instrucciones para el estudiante
1. Lee `docs/propuesta.md` y completa los apartados solicitados.
2. Define tu escenario en `docs/caso_de_uso.md`.
3. Ajusta tu organización en `docs/estructura_repositorio.md`.
4. Planea validaciones en `docs/plan_de_pruebas.md`.
5. Implementa una versión mínima en `src/main.py`.
6. Ejecuta con `bash scripts/run.sh`.
7. Revisa ideas de prueba en `tests/test_plan.md`.

## Criterios de evaluación sugeridos
- Claridad de la propuesta (25%)
- Viabilidad técnica del alcance (25%)
- Calidad de la estructura del repositorio (20%)
- Cobertura básica de pruebas (20%)
- Ejecución del prototipo mínimo (10%)

## Entrega
Sube tu repositorio con todos los archivos actualizados y evidencia breve de ejecución (captura o salida de terminal).
